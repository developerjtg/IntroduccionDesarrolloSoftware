Desarrollar un software es el proceso de construcción de un sistema utilizando su descripción.

Un “desarrollo” de software no es solo un conjunto de programas de computadora, también involucra la documentación de las diferentes etapas desde su concepción, análisis, diseño, implementación, pruebas y mantenimiento.

Análisis

El análisis es un conjunto de procesos orientado a determinar un “QUE” el cual hace parte de un problema estableciendo los límites, quienes intervienen, los recursos que requieren sin pensar en cómo solucionar el problema.

Cuando no hay un adecuado análisis de las necesidades de un proyecto y no existe una apropiada documentación los proyectos de desarrollo de software no ofrecen una solución y terminan siendo foco de nuevos problemas.

Diseño

"El proceso de definición de la arquitectura, componentes, interfaces y otras características de un sistema o componente que resulta de este proceso" [IEEE610.12-90].

Involucra la selección de plataforma de desarrollo, cual será la Base de Datos y cuales son las nuevas necesidades de hardware para el producto final.

Implementación

Es el proceso de poner en funcionamiento el “desarrollo” es el siguiente paso, para cumplirlo debemos agotar las fases de análisis y diseño, así como el proceso de CODIFICACIÓN, que implica la creación de los nuevos programas.

Una vez listo el sistema se implementa de acuerdo con un cronograma que se concertó previamente con todos los involucrados, haciendo necesario capacitar antes durante y después de la puesta en funcionamiento del nuevo sistema.

Pruebas

Las pruebas de software involucran estándares que comprueban si un programa “funciona”, hace los que se planteó, cómo hace su función, que tan bueno es el trabajo previo realizado.

Las pruebas de software se realizan antes de la implementación, pero la prueba de fuego ocurre cuando se implementa para que funcione en un ambiente real

Mantenimiento

El Mantenimiento se relaciona con los mecanismos establecidos desde el diseño para que el nuevo sistema permita hacer reparaciones correcciones o cambios (mejoras).

Esta fase es considerada una de las más importantes en el ciclo de vida del software y depende en gran medida de la calidad de la documentación del sistema.
